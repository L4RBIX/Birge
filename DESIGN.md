# Design

## Foundation
Birge preserves the existing Farfetch-inspired premium commerce foundation. The site should still read as a luxury marketplace at first glance: white surface, black typography, centered premium wordmark, editorial hero rhythm, spacious product rails, large product photography, disciplined grids, and restrained UI chrome.

## Visual Contract
- Do not redesign from scratch.
- Do not remove existing visual assets, local fonts, spacing rhythm, or commerce section patterns unless required for Birge mechanics.
- Prefer adapting existing header, hero, rail, tile, and footer patterns before creating new section structures.
- The correct result is Farfetch-quality premium commerce for Kazakhstan group buying with SIM-backed trust.

## Palette
- Base: white background, black/near-black text, neutral gray dividers.
- Accent: turquoise-green only for group-buy actions, progress, SIM verification, price unlocks, and live demo states.
- Payment cue: Kaspi-like red used only inside the mock payment action.
- No dark theme, neon palette, or excessive gradients.

## Typography
Use the existing local premium fonts and type rhythm. Body/UI remains clean sans-serif; editorial moments may use the existing serif only where the current clone already uses it. Prices use bold, tabular-feeling presentation and must be easy to scan.

## Components
- Header keeps the large centered marketplace wordmark feeling.
- Hero keeps editorial commerce placement and image dominance.
- Deal cards keep product photography first, then compact commerce metadata.
- Main deal detail uses large gallery imagery, clear price waterfall, progress, tiers, escrow notes, and one primary join action.
- Bottom sheets are softly rounded, focused, and mobile-first.
- SIM badges, progress rings/bars, and live toasts are crisp trust mechanics, not decorative stickers.

## Responsive Rules
- Mobile-first at 390px: compact header, swipeable rails, no horizontal page scroll, stable image ratios, 44px touch targets.
- Desktop at 1440px: preserve wide commerce spacing, product rail discipline, and screenshot-ready deal detail composition.

## Avoid
Generic SaaS dashboards, startup card mosaics, abstract illustrations, AI-centered copy, dark mode, heavy dependencies, external image hotlinks, emoji as structural icons, nested cards, layout shifts, and unpolished MVP styling.
